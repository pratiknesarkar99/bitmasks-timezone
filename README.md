# Bit Mask City Finder

A timezone city search tool that demonstrates how bitwise AND operations can replace conventional equality checks for multi-value filtering. Built with vanilla JavaScript (ES Modules), no frameworks, no build step.

---

## What it does

Enter a GMT offset and find every city in that timezone, or every city outside it. Results can be narrowed further by IANA timezone identifier using a pill-style filter. All matching is done at the bit level, not with `===`.

## The core concept

Most timezone lookups look like this:

```js
if (city.gmtOffset === searchOffset) { /* match */ }
```

This app does it differently. Each city is assigned a **bitmask** where a single bit is set at the position corresponding to its GMT offset. Searching means building a mask for the target offset and AND-ing it against each city:

```js
const searchMask = getBitmask(offset);
const match = (city.bitmask & searchMask) !== 0n;
```

For the inverse search (cities outside a timezone), bitwise NOT flips the mask:

```js
const exclusionMask = ~searchMask;
const match = (city.bitmask & exclusionMask) !== 0n;
```

### The formula

Real-world timezone offsets go down to 15-minute increments (e.g. Kathmandu is GMT+5:45, stored as `5.75`). To support this, each offset is mapped to a quarter-hour slot:

```
bitPosition = (gmtOffset * 4) + 56
bitmask = 1n << BigInt(bitPosition)
```

Multiplying by 4 converts to quarter-hour slots. Adding 56 shifts everything positive so GMT-14 lands at slot 0 and GMT+14 at slot 112. JavaScript's native bitwise operators are limited to 32 bits, so **BigInt** is used throughout to handle the full 112-bit range.

## Getting started

### 1. Generate the city dataset

Download the following two files from [GeoNames](https://download.geonames.org/export/dump/):

- `cities15000.zip` (unzip to get `cities15000.txt`)
- `timeZones.txt`

Place both files in the project folder, then run:

```bash
node build-cities.js
```

This produces `cities.json` with ~33,000 cities, each with a name, GMT offset, population, and IANA timezone ID. You only need to run this once.

### 2. Start a local server

ES Modules and `fetch` require HTTP. Open `index.html` directly via `file://` will not work.

```bash
npx serve .
# or
python3 -m http.server 3000
```

Then open `http://localhost:3000` in your browser.

## Features

- Search cities by GMT offset, including half-hour (`5.5`) and quarter-hour (`5.75`) offsets
- Two search modes: cities **in** a timezone or **excluding** a timezone
- Pill-style filter by IANA timezone ID (e.g. `Asia · Kolkata` vs `Asia · Colombo`)
- "Show more / show less" for offsets with more than 10 IANA zones
- Paginated results, 100 cities per page, with prev/next navigation
- Dataset stats panel showing total cities, unique offsets, offset range, and busiest timezone

---

## Data source

City data comes from [GeoNames](https://www.geonames.org/) (`cities15000.txt`), which includes all cities with a population over 15,000. Timezone offsets use the `rawOffset` column from GeoNames' `timeZones.txt`, which reflects standard time without DST adjustments.

**Note:** Offsets are static and do not account for Daylight Saving Time. A city like London appears at GMT+0:00 year-round regardless of BST.

## Key files explained

### `data.js`

Fetches `cities.json` at startup and attaches a BigInt bitmask to each city using the quarter-hour slot formula. Exports `loadCities()` (async) and `getBitmask()`.

### `search.js`

Two exported functions, both accept the loaded cities array and a GMT offset string:

- `searchByOffset(cities, gmtOffset)` — returns cities whose bitmask AND the search mask is non-zero
- `searchExcludingOffset(cities, gmtOffset)` — returns cities whose bitmask AND the bitwise NOT of the search mask is non-zero

Validation rejects offsets outside -14 to +14 and any value that is not a quarter-hour increment.

### `main.js`

Handles all DOM interaction. Cities are loaded once on startup and held in memory. All searching, filtering, and pagination happens in memory with no additional fetches after the initial load.

### `build-cities.js`

A one-time Node script that reads `cities15000.txt` and `timeZones.txt`, joins them on the IANA timezone ID, and writes `cities.json`. Not needed at runtime.

## Extending this project

A few directions worth exploring:

- **Filter by population** — `cities.json` already includes the `population` field. A slider could let users narrow results to major cities only.
- **Sort results** — currently results are sorted alphabetically as they come from GeoNames. Sorting by population descending would surface the most relevant cities first.
- **DST awareness** — `timeZones.txt` includes a `dstOffset` column. With the current date, the app could apply the correct offset dynamically instead of using `rawOffset` year-round.
- **Backend API** — for a production version, the bitmask logic could move to a server with a proper timezone database, removing the need to ship 33k cities to the browser.
